# AGENTS.md

## Projekt
Repozytorium: `atlas-lyr-html`

## Stan bazowy
- Pracuj na aktualnym stanie repo.
- Nie cofaj zaakceptowanych zmian bez wyraźnej prośby użytkownika.
- Nie wracaj do zamkniętych tematów bez wyraźnej potrzeby.

## Zasady pracy
- Pracuj tylko w repo `atlas-lyr-html`.
- Trzymaj się ściśle zakresu zadania.
- Jedno zadanie naraz.
- Jedna zmiana naraz.
- Rób możliwie małe, precyzyjne zmiany.
- Zachowuj spójność z istniejącym stylem projektu.
- Jeśli wystarczy mała poprawka, nie przebudowuj całego pliku.

## Czego nie wolno
- Nie rób `git push`.
- Nie rób commitów.
- Nie twórz branchy bez wyraźnej prośby.
- Nie usuwaj plików bez wyraźnej zgody użytkownika.
- Nie wychodź poza repo.
- Nie zmieniaj rzeczy poza zakresem promptu.
- Nie ruszaj `package.json` ani `server.js`, jeśli zadanie tego nie wymaga.
- Nie psuj wcześniej zaakceptowanych elementów projektu.

## Preferencje użytkownika
- Użytkownik jest początkujący technicznie.
- Zmiany mają być proste, czytelne i przewidywalne.
- Nie komplikuj bez potrzeby.
- Nie proponuj ponownie rzeczy już wykonanych.
- Nie zmuszaj użytkownika do odtwarzania historii projektu.
- Jeśli coś nie wymaga zmiany plików, napisz to jasno i nic nie zmieniaj.

## Zasady dla zmian w UI
- Zachowuj istniejącą estetykę projektu.
- Nie zmieniaj układu wielu stron naraz, jeśli zadanie dotyczy jednej rzeczy.
- Menu, aktywna zakładka i układ strony mają pozostać spójne.
- Przy poprawkach wizualnych trzymaj się dokładnie wymagań użytkownika, bez własnych interpretacji.
